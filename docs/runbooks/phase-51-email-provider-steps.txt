PHASE 51 — TRANSACTIONAL EMAIL ON YOUR OWN DOMAIN (RESEND)
===========================================================

What this does
--------------
Replaces the Phase 47 personal-Gmail SMTP with Resend, a transactional
email provider, sending from your own domain (e.g. noreply@stemquest.app).
Invite links, password resets, and announcement emails then arrive from
your domain with proper SPF/DKIM authentication instead of a personal
Gmail address. Code-wise nothing changes — it is only env vars on the
Render service (dashboard is the source of truth since Phase 48).

Prerequisites: you own a domain and can edit its DNS records
(Cloudflare DNS assumed below).

STEP 1 — Create the Resend account and add your domain
------------------------------------------------------
1. Sign up at https://resend.com (free tier: 3,000 emails/month, 100/day
   — far above what invites + resets need).
2. Resend dashboard -> Domains -> Add Domain -> enter your domain,
   e.g. stemquest.app (or a subdomain like mail.stemquest.app).
3. Resend shows DNS records to add. Keep this tab open.

STEP 2 — Add the DNS records (SPF + DKIM)
-----------------------------------------
In Cloudflare -> your domain -> DNS -> Records, add exactly what Resend
listed. Typically:

  - TXT   send.<domain>      "v=spf1 include:amazonses.com ~all"   (SPF)
  - MX    send.<domain>      feedback-smtp.us-east-1.amazonses.com (10)
  - TXT   resend._domainkey  p=<long DKIM key>                     (DKIM)

Set them as "DNS only" (grey cloud), not proxied.

Back in Resend, click Verify. Propagation is usually minutes; retry if
it fails at first. Status must show Verified before continuing.

STEP 3 — Create the SMTP credential
-----------------------------------
Resend dashboard -> API Keys -> Create API Key
  - Name: stemquest-render-smtp
  - Permission: Sending access only
Copy the key (shown once). Store it in your password manager.

STEP 4 — Flip the Render env vars
---------------------------------
Render dashboard -> stemquest-api-va -> Environment. Change ONLY these
(leave EMAIL_BACKEND, EMAIL_PORT, EMAIL_USE_TLS as they are):

  EMAIL_HOST           smtp.resend.com
  EMAIL_HOST_USER      resend
  EMAIL_HOST_PASSWORD  <the API key from step 3>
  DEFAULT_FROM_EMAIL   STEM Quest <noreply@your-domain>

The from-address domain MUST be the domain you verified in step 2.
Save -> the service restarts with the new settings.

STEP 5 — Send a test
--------------------
1. Log into the live site as the instructor account.
2. Roster page of a course -> Invite students -> your own personal email.
3. The invite should arrive within a minute, From: STEM Quest
   <noreply@your-domain>. In Gmail use "Show original" and check
   SPF: PASS and DKIM: PASS.
4. Click the link and walk the accept flow end to end (this doubles as
   the Phase 51 manual verification).

If nothing arrives: Resend dashboard -> Emails shows every send attempt
and its error; also check the Render service logs for SMTP errors.

Rollback
--------
Set the four env vars back to their previous Gmail values (Render keeps
the old values in the env-var edit history) and save. The Phase 47
runbook (phase-47-email-rollout.txt) documents the Gmail setup.

Cleanup once verified
---------------------
- Revoke the old Gmail app password (Google account -> Security -> App
  passwords) so the personal-Gmail path is fully retired.
